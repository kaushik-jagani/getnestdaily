<p>Something unusual is happening in the tech world — CEOs are going back to coding.</p><p>For years, top executives moved away from writing code as their responsibilities shifted toward strategy, management, and scaling organizations. But with the rise of AI coding tools, that trend is reversing.</p><h2>CEOs Are Back in the Codebase</h2><p>Recent reports suggest that Mark Zuckerberg submitted code to Meta's internal systems for the first time in nearly two decades. Similarly, Brian Armstrong shared his GitHub activity, highlighting a return to hands-on coding.</p><figure class="post-image"><img src="assets/images/posts/ai-bringing-ceos-back-to-coding/brian-armstrong-tweet.jpg" alt="Brian Armstrong tweet showing GitHub contributions - CEO back in the codebase thanks to AI" loading="lazy"><figcaption>Brian Armstrong (@brian_armstrong) sharing his GitHub activity — CEOs are returning to the codebase thanks to AI.</figcaption></figure><p>Other leaders are following the same path. Garry Tan has resumed coding after many years, and Shopify's Tobias Lütke has experimented with AI-driven code optimizations.</p><p>This isn't just symbolic. AI tools are lowering the barrier to entry, allowing even those who stepped away from coding to contribute again.</p><h2>What Changed? The Role of AI</h2><p>AI-powered coding assistants can now generate, refactor, and optimize code quickly. Instead of writing everything from scratch, developers can guide AI tools and review their outputs.</p><p>This shift means coding is no longer just about typing syntax — it's about understanding systems, identifying problems, and verifying solutions.</p><h2>The Real Shift: From Writing Code to Judging Code</h2><p>The biggest change is not who can write code — it's who can evaluate it.</p><p>AI can produce working code, but it can also generate inefficient, insecure, or incorrect solutions. This makes human judgment more important than ever.</p><p>In many organizations, engineers are increasingly expected to review, validate, and improve AI-generated code rather than create everything manually.</p><h2>What This Means for Developers</h2><p>If you're a developer, this shift has real implications for how you build your career and where you invest your learning time.</p><ul><li>Understanding fundamentals becomes more valuable than memorizing syntax.</li><li>Debugging and code review skills will be critical.</li><li>System design and architecture knowledge will matter more.</li><li>Communication and decision-making will become key differentiators.</li><li>Domain expertise — understanding the business problem you're solving — will set you apart from AI-generated code.</li></ul><p>In simple terms, your value will come from knowing whether code is correct — not just writing it. The developers who thrive will be those who can look at AI-generated output and quickly identify edge cases, security vulnerabilities, and performance bottlenecks that the AI missed.</p><h2>Is Coding Really 'Solved'?</h2><p>Some industry voices claim that coding is becoming a solved problem. While AI has made huge progress, this statement is still debatable.</p><p>Real-world software involves complex trade-offs, edge cases, and long-term maintenance challenges that require deep contextual understanding. AI can assist with boilerplate code and common patterns, but it cannot fully replace human reasoning and accountability.</p><p>Consider the challenges that still require human expertise: navigating ambiguous product requirements, making architectural decisions that affect years of future development, handling sensitive data with proper security practices, and coordinating across teams with competing priorities. These are areas where AI serves as a tool, not a replacement.</p><h2>Final Thoughts</h2><p>The return of CEOs to coding is not just a trend — it's a signal.</p><p>AI is changing the nature of software development. It's making coding more accessible, but also raising the bar for what it means to be a skilled developer.</p><p>The question is no longer: &ldquo;Can you write code?&rdquo;</p><p>The real question is: &ldquo;Do you know if the code is right?&rdquo;</p>
