# TrendBriefly — Copilot Instructions

Before making ANY changes to this project, **read the implementation guide first**:

📄 `docs/IMPLEMENTATION.md`

This file contains:
- Full project architecture and folder structure
- JS module load order and dependencies
- Path resolution system (ROOT_PATH, PAGES_PATH, BLOG_PATH)
- posts.json schema and rules
- Step-by-step procedures for adding posts, authors, categories, and pages
- Styling conventions and CSS variable system
- SEO checklist
- Common pitfalls and fixes

## Critical Rules

1. **Always read `docs/IMPLEMENTATION.md` before implementing anything.**
2. Posts data lives in `data/posts.json` — single source of truth.
3. Blog post URLs are clean: `/blog/{slug}/` not `?slug=`.
4. Image paths in `posts.json` and content are relative to root (no leading `../`).
5. JS uses global scope — `config.js` must load first in all HTML files.
6. After changing `posts.json`, bump `POSTS_CACHE_KEY` version in `js/config.js`.
7. New posts require: folder in `blog/`, images in `assets/images/posts/`, entry in `posts.json`, sitemap update.
8. Use `node scripts/new-post.js "slug"` to scaffold new posts.
